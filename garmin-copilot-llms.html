Habit 01

Plan mode

Ask for the plan before the code. You catch wrong assumptions while they're cheap — and get a diff you can actually review.

Rule of thumb: the bigger the change, the more it's worth planning first.
plan-mode
# proposed steps
1. read auth module
2. add token refresh
3. update tests

# approve? (y/n)
GARMIN
Habit 02

Pick the right model

Not every task needs the biggest model. Reach for a fast one to iterate; reach for a deep one when the problem is genuinely hard.

Match effort to stakes — trivial edits and hard architecture are not the same job.
Fast
Iterate
  • Boilerplate & edits
  • Quick questions
  • Tight feedback loops
Deep
Reason
  • Thorny bugs
  • Architecture
  • Multi-file changes

Habits 03 & 04

Mind the context window

The model only knows what's in its context. Keep it focused, let it compact when it fills, and structure prompts to land cache hits.

Stable prefix = cache hit — cheaper, faster turns. Curate what it sees; drop what it doesn't need.

SECTION 05

Teach your
harness

Skills & Agents.md
GARMIN
Loaded by prompt & context

Skills

Reusable workflows the agent pulls in when your prompt calls for them. A Skill is the interface to your component — how to build it, test it, work with it.

Shareable by design — one person's workflow becomes everyone's.
.copilot/skills/
# prompt: "release the maps SDK"
→ loads  maps-sdk-release/
→ loads  changelog-writer/
# right knowledge, right moment
GARMIN
Loaded by location

Agents.md

Instructions the agent reads based on where it's working in the repo. Drop one in a directory and every agent that touches it inherits your local rules.

Local & private — the conventions that live with your code.
repo tree
/AGENTS.md  // house rules
/firmware/AGENTS.md  // build quirks
/mobile/AGENTS.md  // lint + test
# reads the one nearest the work
GARMIN
When to reach for which

Two ways to teach the loop

Skills
Shareable workflows
  • Triggered by your prompt & context
  • An interface to your component
  • Team knowledge, packaged once
  • Use for anything worth sharing
Agents.md
Local conventions
  • Triggered by where work happens
  • Lives next to the code it governs
  • Private, repo-specific rules
  • Use for local, personal workflows
